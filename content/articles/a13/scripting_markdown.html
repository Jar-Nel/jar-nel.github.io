Markdown for your Markup

Markdown was created as a simple way for non-programmers to write in a readable format that could be easily coneverted into HTML.

Markdown uses a simple formatting syntax. To format text, Markdown uses punctuation and characters most people are already familiar with.

Many GitHub users make use of Markdown for their documentation. It's ability for quick and simple formatting along with a human readibility of the raw text makes it well suited for this purpose.

Here are some common Markdown styles

  • Headers
    • # H1
    • ## H2
    • ### H3
    • #### H4
  • Emphasis
    • Italics: *italics* or _italics_
    • Bold: **bold** __bold__
    • Strikethrough: ~~Strikethrough~~
  • Lists
    • Ordered lists just use numbers 1., 2., 3.
    • Unordered lists use *, -, or +
  • Images
    • Inline: ![alt text](https://urlofimage "Image alt text")
    • Reference: ![alt text][image_1]
      [image_1]: https://urlofimage "Image alt text"
