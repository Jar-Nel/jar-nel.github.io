If you were paying attention to the URL field in your browser as you navigate this site you may have noticed that the pages are requested a little differently. For example, the URL for this article includes the text PageLoader.html#article.html~a17. You might be wondering what this does and why it looks like this. On this page I will make an attempt to explain it.

The request PageLoader.html#article.html~a17 is broken into three pieces. The first part, PageLoader.html, requests the page that has all the menu navigation, CSS themes, and JavaScript content for the site. Once the resources for PageLoader.html have been loaded into the browser, JavaScript takes over. The next part, article.html~a17, instructs the JavaScript that we want to load the article.html content template and populate it with the data for article a17, which is the article youâ€™re reading now. The hash symbol (#) is used to separate the content from the request. I chose the hash instead of a more traditional query string because it can be modified without causing the browser to reload the page. This way I can support deep linking directly to the content but prevent reused assets from reloading and redrawing as the site is navigated after the initial load. It also means all of those reused assets are loaded from a single location, which allows me to change something like the menu structure in one place and have that change appear on every page the site loads.
